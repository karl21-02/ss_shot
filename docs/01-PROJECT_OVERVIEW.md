## 2. 배경 및 문제 정의

### 2.1 현재 상황 (As-Is)

스마트폰 사용자들은 매일 수많은 스크린샷을 찍습니다:
- 계좌 이체 확인
- 배송 조회
- 이벤트 정보
- 중요한 대화 내용
- 영수증 및 쿠폰

하지만 이 스크린샷들은 **갤러리에 방치**되어:
- 필요할 때 찾기 어려움
- 갤러리 용량만 차지
- 중요한 정보가 묻혀버림

### 2.2 목표 상황 (To-Be)

SS-Shot을 통해:
- **검색 한 번**으로 원하는 스크린샷을 즉시 찾음
- **AI가 자동 분류**하여 체계적으로 정리
- **게임처럼 재미있게** 불필요한 스크린샷 정리

---

## 3. 핵심 가치 (Core Value Proposition)

### 3.1 Find Instantly (즉시 검색)

> "텍스트 검색으로 0.1초 만에 이미지를 찾는다"

- OCR 기술로 스크린샷 내 모든 텍스트 추출
- Full-Text Search로 초고속 검색
- 검색어 위치 하이라이팅

### 3.2 Organize Automatically (자동 분류)

> "AI가 금융, 쇼핑, 일정 등으로 알아서 분류해 준다"

- 키워드 기반 자동 카테고리 분류
- 사용자 패턴 학습 (향후 확장)
- 직관적인 탭 기반 필터링

### 3.3 Clean Fun (재미있는 정리)

> "지루한 정리를 게임처럼 즐긴다"

- Tinder 스타일 스와이프 UI
- 햅틱 피드백과 애니메이션
- 정리 통계 및 성취감 제공

---

## 4. 타겟 사용자 (Target User)

### 4.1 Primary Persona

**직장인 김민수 (32세)**
- 하루 평균 5~10장의 스크린샷 생성
- 갤러리에 1,000장 이상의 스크린샷 보유
- "저번에 찍어둔 계좌번호가 어디 있지?" 경험 빈번
- 정리하고 싶지만 시간이 없음

### 4.2 Secondary Persona

**대학생 이서연 (23세)**
- 쇼핑몰 할인 정보, 이벤트 캡처 다수
- SNS 대화 캡처 습관
- 갤러리 용량 부족 경험
- 깔끔한 UI/UX에 민감

## 5. 성공 지표 (KPI)

### 5.1 핵심 지표

| 지표 | 목표 | 측정 방법 |
| --- | --- | --- |
| **DAU** | 출시 3개월 후 10,000명 | Firebase Analytics |
| **검색 사용률** | DAU의 60% 이상 | 검색 API 호출 수 |
| **정리 모드 완료율** | 진입자의 70% 이상 | 정리 완료 이벤트 |
| **리텐션 (D7)** | 40% 이상 | Cohort 분석 |

### 5.2 기술 지표

| 지표 | 목표 | 측정 방법 |
| --- | --- | --- |
| **검색 응답 속도** | 200ms 이내 | 서버 로그 |
| **OCR 처리 속도** | 이미지당 500ms 이내 | 클라이언트 로그 |
| **앱 크래시율** | 0.1% 미만 | Firebase Crashlytics |

## 6. 개발 마일스톤 (Milestones)

### Phase 1: 기반 구축

**목표:** 기본 인프라 및 데이터 파이프라인 구성

- Spring Boot 프로젝트 세팅
- AWS RDS (MySQL) 설계 및 구축
- Flutter 프로젝트 초기화
- 갤러리 접근 및 이미지 로드

**핵심 산출물:**
- 서버 Hello World API
- Flutter 갤러리 그리드 UI

### Phase 2: 코어 연동

**목표:** OCR 및 동기화 기능 완성

- Google ML Kit OCR 연동
- `/api/sync` API 개발
- 메타데이터 저장 및 조회
- 네트워크 실패 시 재시도 로직

**핵심 산출물:**
- 스크린샷 OCR 결과 서버 저장
- 동기화 상태 UI 표시

### Phase 3: 기능 구현

**목표:** 핵심 사용자 기능 완성

- 검색 API (Full-Text Search)
- 자동 카테고리 분류 로직
- 스마트 액션 버튼 (복사, 열기 등)
- 하이라이팅 기능

**핵심 산출물:**
- 검색 결과 화면
- 카테고리 탭 필터링
- 상세 뷰어 스마트 액션

### Phase 4: 고도화

**목표:** 사용자 경험 완성 및 폴리싱

- Swipe-to-Clean 정리 모드
- 소셜 로그인 (Google, Apple)
- 디자인 시스템 적용
- 애니메이션 및 마이크로 인터랙션

**핵심 산출물:**
- 정리 모드 UI 완성
- 로그인/온보딩 플로우
- 최종 디자인 적용
